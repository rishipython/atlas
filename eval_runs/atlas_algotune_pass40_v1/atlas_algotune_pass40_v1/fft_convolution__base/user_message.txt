Problem: fft_convolution

AlgoTune: speed up 1-D convolution.

Given two 1-D float64 numpy arrays ``x`` (length ``N``) and ``h`` (length
``M``), return their full linear convolution, i.e. an array ``y`` of
length ``N + M - 1`` where::

    y[n] = sum_k x[k] * h[n - k]

The reference implementation is a correct-but-very-slow double-loop
direct-sum in pure Python. Your task is to replace it with an
implementation that returns a numerically-equal result on the evaluator's
test inputs but runs far faster. FFT-based convolution
(``scipy.signal.fftconvolve`` or a numpy-based FFT trick) gives the
biggest speedups for large inputs.

Reference (correct but slow) implementation — replace its body with a faster implementation, keeping the same function name and signature:
```python
import numpy as np


def conv1d(x: np.ndarray, h: np.ndarray) -> np.ndarray:
    """Return ``np.convolve(x, h, mode='full')`` but MUCH faster."""
    N = x.shape[0]
    M = h.shape[0]
    out = np.zeros(N + M - 1, dtype=np.float64)
    for n in range(N + M - 1):
        s = 0.0
        kmin = max(0, n - M + 1)
        kmax = min(N - 1, n)
        for k in range(kmin, kmax + 1):
            s += x[k] * h[n - k]
        out[n] = s
    return out
```
