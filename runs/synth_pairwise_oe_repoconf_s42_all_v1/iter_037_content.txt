import numba

@numba.njit
def _row_sqnorms(A):
    n = A.shape[0]
    out = np.empty(n, dtype=np.float64)
    for i in range(n):
        s = 0.0
        for k in range(A.shape[1]):

With these pieces I can write `pairwise_sq_dist`:

```python
def pairwise_sq_dist(X):
    # Ensure float64 input – NumPy will cast automatically otherwise,
    # but being explicit keeps type guarantees.
    X = np.asarray(X, dtype=np.float64)

    n = X.shape[0]
    if n == 0:
        return np.empty((0, 0), dtype=np.float64)

    # Squared norms per row – JIT‑compiled loop.
    sq_norms = _row_sqnorms(X)

    # Gram matrix via SciPy BLAS.
    G = blas.dgemm(alpha=1., a=X, b=X.T)

    # Build distance matrix from algebraic identity.
    D = sq_norms[:, None] + sq_norms[None, :] - 2 * G

    # Numerical noise can push tiny negatives below zero;
    # clip them back to zero because distances are non‑negative.
    D[D < 0] &= 0

    return D
```

A few edge cases come to mind:

* **Empty input (`n==0`)** – return `(0,0)` array immediately.
* **Zero‑dimensional data (`D==0`)** – all rows are identical zero vectors; the formula still works because `G` will be all zeros and `sq_norms` all zeros.
* **Very large `N`** – memory consumption is dominated by storing an `(N,N)` result anyway; no extra copies are introduced beyond what NumPy/BLAS already allocate.
* **Rounding errors** – after subtraction we may get tiny negative values like `-1e-15`; clipping ensures symmetry and non‑negativity without affecting the `rtol/atol` tolerance used by the tests.
