We need a fast way to compute the full pairwise squared Euclidean distance matrix without looping in Python – vectorised math plus BLAS is the usual route.  
The classic identity is  

`||x−y||² = ||x||² + ||y||² − 2·x·yᵀ`.  

So if we pre‑compute the row‑wise norms (`sq`) and the Gram matrix (`G`) via an efficient BLAS routine (`dgemm` from SciPy), assembling the distance matrix is just a few broadcasting operations that stay on the CPU core level and avoid any explicit Python loops.

We also have to satisfy the brief’s “use SciPy” and “use Numba” constraints while keeping everything vectorised from the user’s perspective – no nested Python loops should appear in normal execution time.  
A clean way is to put the norm computation into a tiny Numba‑JITed routine so that part runs on machine code while still being called from plain Python code that orchestrates everything else with SciPy’s BLAS backend.

Here’s how it all fits together:

```python
import numpy as np
import scipy.linalg.blas as blas
import numba

# A tiny JIT‑compiled helper – it does nothing fancy,
# just sums squares along rows using explicit loops that are compiled away.
@numba.njit(parallel=False)
def _row_squares(X):
    n, d = X.shape
    out = np.empty(n, dtype=np.float64)
    for i in range(n):
        s = 0.0
        for j in range(d):
            v = X[i, j]
            s += v * v
        out[i] = s
    return out

def pairwise_sq_dist(X):
    """
    Return an (N,N) array where entry [i,j] is ||X[i]-X[j]||².
    
    Parameters:
        X : ndarray of shape (N,D), dtype float64
    
    Returns:
        ndarray of shape (N,N), dtype float64
    """
    
    # Ensure proper type – NumPy will cast ints etc.
    X_arr = np.asarray(X, dtype=np.float64)

    # Quick path for empty input – avoid BLAS calls on size‑zero arrays.
    if X_arr.size == 0:
        return np.empty((0, 0), dtype=np.float64)

    # Row‑wise squared norms – computed by Numba so no Python overhead.
    sq_norms = _row_squares(X_arr)

    # Gram matrix via SciPy BLAS – highly optimised.
    G = blas.dgemm(alpha=1.0,
                   a=X_arr,
                   b=X_arr.T,
                   trans_a=False,
                   trans_b=False)

    # Assemble distances using broadcasting.
    Dmat = sq_norms[:, None] + sq_norms[None, :] - 2 * G

    # Numerical quirks can leave tiny negatives when points coincide;
    # clip them down to exact zero – keeps allclose happy.
    np.maximum(Dmat, 0., out=Dmat)

    return Dmat
```
