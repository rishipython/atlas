We need an efficient pairwise squared Euclidean distance routine without any Python‑level loops – everything has to be vectorised or compiled away by NumPy/BLAS/Numba/Scipy under the hood.
Let’s start from the classic Gram‑matrix trick:
```
D[i,j] = ||x_i||² + ||x_j||² – 2·x_i·x_jᵀ .
```
The heavy lifting is the dot product `X @ Xᵀ`.  
If I just write `G = X @ X.T` I’ll get NumPy’s own BLAS call – good enough,
but the brief explicitly wants us to “use **scipy** functions somewhere”.  
Scipy offers a thin wrapper around BLAS (`scipy.linalg.blas`), so I’ll replace
the NumPy dot with `scipy.linalg.blas.dgemm`.  
That gives me an explicit matrix multiplication step backed by BLAS,
and I still keep the rest of the computation vectorised:
```
diag   = np.diag(G)
D      = diag[:,None] + diag[None,:] - 2*G
```
Because of floating‑point round‑off there might be tiny negatives (`–1e‑16`)
in `D`.  A single pass setting anything below zero back to zero fixes this.
I’ll put that pass into a small Numba‑compiled routine so it runs at C speed,
and it satisfies “uses **numba** JIT compilation”.

Putting it together:

```python
import numpy as np
import scipy.linalg as la
from numba import njit

@njit
def _clip_negative_to_zero(arr):
    n = arr.shape[0]
    for i in range(n):
        for j in range(n):
            if arr[i,j] < 0:
                arr[i,j] = 0

def pairwise_sq_dist(X):
    """
    Return the (N,N) squared Euclidean distance matrix.
    
    Parameters
    ----------
    X : ndarray of shape (N,D), dtype float64
    
    Returns
    -------
    D : ndarray of shape (N,N), dtype float64
        D[i,j] == sum_k (X[i,k]-X[j,k])**2 .
    """
    # Matrix multiplication via Scipy’s BLAS wrapper.
    G   = la.blas.dgemm(alpha=1., a=X, b=X.T)
    
    # Gram‑matrix trick.
    d   = np.diag(G)
    D   = d[:,None] + d[None,:] - 2*G
    
    # Remove any tiny negative values caused by round‑off.
    _clip_negative_to_zero(D)
    
    return D
```

The routine has no Python‑level loops – all heavy work goes through BLAS,
NumPy’s vectorised ops or Numba‑compiled inner loops.
It uses `scipy.linalg.blas` for the core matrix multiply,
`numba.njit` for the tiny clipping pass,
and keeps everything fast while staying numerically close to the reference implementation.
