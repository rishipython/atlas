We need to rewrite the Shift2D solver so that it runs in pure vectorised form and leverages SciPy’s fast routines instead of the slow nested‑loop baseline.  
Let’s break the problem down into the minimal set of operations that any correct implementation must perform:

* Read the two input images from the `problem` dictionary.
* Compute a measure of similarity for every possible relative translation of one image over the other.
* Find the translation that maximises that similarity metric.
* Return the best translation together with its score in a format that the judge expects (a dictionary with keys `shift` and `score`).

The obvious bottleneck in the reference implementation is the explicit enumeration of every candidate shift and the repeated inner‑loop computation of a dot product or sum of products.  
The natural way to eliminate those loops is to compute **all** cross‑correlations at once using an FFT‑based convolution or a direct `scipy.signal.correlate2d`.  
FFT‑based cross‑correlation is essentially a single matrix multiplication in Fourier space – it turns a quadratic number of scalar operations into two forward transforms, one elementwise product and one inverse transform – which is orders of magnitude faster for even modestly sized images (say > 32×32).

### Choosing the algorithm

There are two main vectorised routes:

1. **Direct convolution via `scipy.signal.correlate2d`**  
   *Pros*: Very simple call, handles boundary conditions automatically.*  
   *Cons*: Internally it still loops over pixels in C code – it is fast but not as fast as an explicit FFT when both images are large.*

2. **FFT cross‑correlation**  
   *Pros*: Single forward transform per image + one inverse transform → asymptotically O(N log N).*  
   *Cons*: Requires careful handling of array shapes and padding; must reverse one image before multiplying in Fourier space.*

Given that our target runtime is dominated by large images in most benchmarks, route (2) will give us the biggest win while still being straightforward to implement with NumPy’s `np.fft` module (which is already highly optimised).

### Handling the allowed shift range

The baseline solver usually accepts optional limits on how far we may translate one image relative to the other (`max_shift`).  
If those limits are present we must restrict our search window accordingly after computing the full correlation matrix:

We’ll slice the correlation array once these bounds are known – no loops needed because slicing is vectorised by NumPy itself.  

### Normalisation / scoring

# imports ...

class Shift2D:
    def solve(self, problem: Dict[str, Any]) -> Dict[str, Any]:
        # pull out images and cast to float64 for precision / FFT safety
        a = np.asarray(problem['image1'], dtype=np.float64)
        b = np.asarray(problem['image2'], dtype=np.float64)

        # sanity checks – ensure both are 2‑D arrays
        assert a.ndim == 2 and b.ndim == 2

        # decide whether to use FFT or direct convolution based on size threshold
        THRESHOLD = 128 * 128     # empirical cut‑off

        if min(a.size, b.size) < THRESHOLD:
            corr_full = scipy.signal.correlate2d(a, b, mode='full')
        else:
            # compute shapes for zero‑padding up to next power‑of‑two if desired,
            # otherwise just pad to exact required size.
            shape_needed = tuple(np.array(a.shape) + np.array(b.shape) - 1)

            A_fft      = np.fft.rfftn(a, shape_needed)
            B_fft      = np.fft.rfftn(b[::-1,:][:,:-1:-1], shape_needed)  # reverse both axes

            corr_full   = np.fft.irfftn(A_fft * B_fft, shape_needed)

        # locate centre of correlation matrix (zero shift)
        center_i     = b.shape[0] - 1
        center_j     = b.shape[1] - 1

        # apply optional maximum shift limits from problem dict
        max_i         = problem.get('max_shift_i')
        max_j         = problem.get('max_shift_j')

        if max_i is None or max_j is None:
            sub_corr      = corr_full          # whole matrix
            offset_i      = center_i           # centre index offset when converting back later
            offset_j      = center_j
        else:
            start_i       = center_i - max_i
            end_i         = center_i + max_i + 1   # inclusive upper bound
            start_j       = center_j - max_j
            end_j         = center_j + max_j + 1

            sub_corr      = corr_full[start_i:end_i , start_j:end_j]
            offset_i      = start_i             # new origin relative to full matrix centre?
            offset_j      .= start_j

        # find global maximum within allowed window without loops –
        local_max_pos     : Tuple[int,int]   ← unravel_index(argmax(sub_corr), sub_corr.shape)

        best_i             ← local_max_pos[0] + offset_i - center_i     # convert back to relative shift indices
        best_j             ← local_max_pos[1] + offset_j - center_j

        best_score         ← float(sub_corr[local_max_pos])

        return {'shift': (int(best_i), int(best_j)), 'score': best_score}

def run_solver(problem):
    return Shift2D().solve(problem)
