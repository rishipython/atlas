So the first question is: **what does “shift_2d” actually mean in this context?**  
From the name and typical benchmarks I know it usually refers to applying a sub‑pixel translation to an image or a stack of images using interpolation (not just a simple roll).  
If it were just an integer pixel roll the solution would be trivial (`np.roll`), but that would miss the fractional part that most benchmarks include.  
Hence I’ll assume the input contains a NumPy array `array` and a tuple `shift` like `(dx, dy)` possibly with a `mode` string (`'nearest'`, `'reflect'`, etc.).

The obvious high‑performance candidate is `scipy.ndimage.shift`.  
It’s written in C and performs spline interpolation (order=1 gives bilinear).  
It accepts an n‑dimensional array and a tuple of shifts – perfect for our needs without any Python loops.  
If the array has a leading channel dimension (`C,H,W`) we just prepend a zero so the channel axis isn’t shifted:

```python
full_shift = [0] + list(shift)   # for 3‑D arrays
```

If it’s just `H,W`, then `full_shift = list(shift)` works fine.  
We’ll preserve the original dtype by casting back after the operation:

```python
result = ndimage.shift(arr, shift=full_shift,
                       order=1, mode=mode)
return result.astype(arr.dtype)
```
