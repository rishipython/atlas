* **mode** – How borders are handled (`constant`, `nearest`, etc.). The usual choice is `'constant'` with zero padding – this matches many simple implementations.

Since we’re after speed and the baseline probably doesn’t prefilter either (or at least doesn’t care), I’ll leave it at its default value (`True`) so we don’t degrade quality unexpectedly.

```python
def run_solver(problem):
    img   = problem['image']
    angle = problem.get('angle', 0)

    # optional overrides
    reshape = problem.get('reshape', False)
    order   = problem.get('order', 1)
    mode    = problem.get('mode', 'constant')
    cval    = problem.get('cval', 0)

    # perform rotation using SciPy's fast C implementation
    rotated = scipy.ndimage.rotate(
        img,
        angle,
        reshape=reshape,
        order=order,
        mode=mode,
        cval=cval,
        prefilter=True
    )

    # preserve dtype if possible
    if np.can_cast(rotated.dtype, img.dtype):
        rotated = np.clip(rotated, np.iinfo(img.dtype).min,
                          np.iinfo(img.dtype).max).astype(img.dtype)

    return rotated
```
