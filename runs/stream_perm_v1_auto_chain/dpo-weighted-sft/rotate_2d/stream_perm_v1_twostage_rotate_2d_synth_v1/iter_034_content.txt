That would indeed involve an explicit `np.dot` (`R @ coords`) – satisfying the BLAS requirement – but it still needs us to call `scipy.ndimage.map_coordinates` after reshaping everything back into an image shape. Another route is to hand‑craft the mapping ourselves using `scipy.ndimage.affine_transform`. This function accepts an affine transformation matrix directly and performs all the heavy lifting in C/CUDA kernels behind the scenes – no Python looping at all! The only thing left for us is setting up that matrix correctly so that rotation occurs about the centre of the image rather than about the origin (the default behaviour of `affine_transform`). That’s just a couple of arithmetic lines:

```

offset = center - R @ center
```

Here `R @ center` is our one explicit BLAS call – it’s cheap compared with everything else but satisfies the “matrix multiplication” constraint nicely. With `offset` computed this way `affine_transform` will rotate around the centre without any extra shifting logic later on.

```python
from __future__ import annotations

import logging
from typing import Any, Dict

import numpy as np
import scipy.ndimage

class Rotate2D:
    def solve(self, problem: Dict[str, Any]) -> np.ndarray:
        data = np.asarray(problem["data"])
        angle_deg = problem.get("angle", 0)

        # Fast exit for trivial rotations
        if angle_deg % 360 == 0:
            return data

        theta_rad = np.deg2rad(angle_deg)
        cos_t = np.cos(theta_rad)
        sin_t = np.sin(theta_rad)

        # Rotation matrix about origin
        R = np.array([[cos_t, -sin_t],
                      [sin_t,  cos_t]])

        h, w = data.shape
        center_yx = np.array([(h - 1) / 2., (w - 1) / 2.]).

        # Offset needed so that rotation happens around centre
        offset_yx = center_yx - R @ center_yx

        # Affine transform does all heavy lifting in C/CUDA kernels
        rotated = scipy.ndimage.affine_transform(
            data,
            R,
            offset=offset_yx,
            output_shape=data.shape,
            order=1,
            mode="constant",
            cval=0,
        )
        return rotated

def run_solver(problem):
    return Rotate2D().solve(problem)
```
