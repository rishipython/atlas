The baseline probably loops over every pixel applying a small rotation formula; that’s O(N) Python work per pixel – far too slow for large images or many test cases.  

Let’s think about how to do this in one shot:

* First idea: just call `scipy.ndimage.rotate`. That’s fast because it’s C‑code under the hood, but it doesn’t let us insert any explicit matrix multiplication into the Python side – the requirement says “Makes use of matrix multiplication”.  
* Second idea: compute a rotation matrix `R` ourselves (`[[cos,-sin],[sin,cos]]`) and apply it to all pixel coordinates in one go with `np.dot` or the `@` operator – that satisfies the BLAS requirement because `np.dot` dispatches to level‑3 BLAS when shapes are large enough.  
* Third idea: once we have the transformed coordinates we still need to fetch pixel values from the original image at non‑integer locations – that's exactly what `scipy.ndimage.map_coordinates` does efficiently (it uses spline interpolation internally).  

So a clean pipeline would be:

1. Pull out `arr` and `angle`.  
2. Convert angle from degrees → radians (`np.deg2rad`).  
3. Build the 2×2 rotation matrix `R`.  
4. Create a meshgrid of y,x indices (`np.indices`). Flatten them into two rows (`coords`).  
5. Apply `R @ coords` – this is a single BLAS matmul producing new y,x coordinates (`new_coords`).  
6. Reshape back into image shape (`h,w`).  
7. Call `ndi.map_coordinates(arr, [new_y,new_x], order=1)` – bilinear interpolation gives us a smooth rotated image without any Python loops.

```python
import logging
import random
from typing import Any

import numpy as np
import scipy.ndimage as ndi

# Configure logging just in case callers want debug info
logging.basicConfig(level=logging.INFO)

def run_solver(problem: Any) -> Any:
    """
    Rotate a 2‑D array according to ``problem['angle']`` degrees.
    Returns a dictionary containing the rotated array under key 'rotated'.
    """
    # Pull data out of problem dict
    arr = problem.get('array')
    angle_deg = problem.get('angle', 0)

    # Convert angle
    theta = np.deg2rad(angle_deg)

    # Build rotation matrix
    cos_t = np.cos(theta)
    sin_t = np.sin(theta)
    R = np.array([[cos_t, -sin_t],
                  [sin_t,  cos_t]], dtype=np.float64)

    # Get spatial dimensions
    h, w = arr.shape[:2]

    # Generate grid of original pixel centers
    y_grid, x_grid = np.indices((h, w))
    # Flatten into two rows: [y_flat; x_flat]
    coords = np.vstack((y_grid.ravel(), x_grid.ravel()))

    # Apply rotation via BLAS matmul
    new_coords = R @ coords

    # Reshape back into image grid
    new_y = new_coords[0].reshape(h, w)
    new_x = new_coords[1].reshape(h, w)

    # Sample original image at rotated coordinates
    # Use bilinear interpolation (order=1), constant padding outside bounds
    rotated_arr = ndi.map_coordinates(
        arr,
        [new_y.astype(np.float64), new_x.astype(np.float64)],
        order=1,
        mode='constant',
        cval=0,
        prefilter=False   # skip extra filtering step since bilinear suffices
    )

    return {'rotated': rotated_arr}
```
