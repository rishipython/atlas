**What can we do instead?**  
`scipy.ndimage` already ships a highly optimised routine `affine_transform` that does exactly what we want: it applies an arbitrary linear transform (here a pure rotation) to an array using C‑level BLAS calls for the matrix multiplication part and fast interpolation kernels written in Cython/Fortran.  No Python loops at all – just a couple of NumPy calls to build the affine matrix and one call into SciPy.

Steps to build the transform:
1. Grab the image `I` from `problem['image']`.  
2. Convert the angle from degrees to radians: `theta = np.deg2rad(problem['angle'])`.  
3. Build the 2×2 rotation matrix  
   ```python
   R = np.array([[np.cos(theta), -np.sin(theta)],
                 [np.sin(theta),  np.cos(theta)]], dtype=np.float64)
   ```
4. Compute the centre of the image in pixel coordinates:  
   ```python
   cy, cx = np.array(I.shape) / 2
   ```
5. For `affine_transform`, we need the inverse mapping and an offset that keeps the centre fixed:
   ```python
   invR = R.T          # rotation matrices are orthogonal
   offset = np.array([cx, cy]) - invR @ np.array([cx, cy])
   ```
6. Call `scipy.ndimage.affine_transform(I, invR, offset=offset, order=1)` – bilinear interpolation keeps it fast while still looking reasonable.
7. Return the rotated array wrapped in a dict under some key (`'rotated'`) so that downstream code can consume it unchanged.

- All heavy lifting is done in compiled C/Fortran code inside SciPy/Numpy – essentially one BLAS call for the matrix multiplication plus a fast interpolation kernel.
