We need a fast way to apply a 2‑D affine warp while staying faithful to the verifier’s expectations – it checks against another call that does exactly the same thing with `scipy.ndimage.affine_transform`.  
The trick is that the verifier’s “reference” actually passes the full \(2\times3\) matrix straight into `affine_transform`, even though the documented signature expects a \(k\times k\) linear part plus an optional offset vector.  
Because both our solver *and* the verifier do this identical misuse, any deviation will make us fail validation even if the numerical result looks correct visually – so we must mimic that behaviour exactly.

### What really hurts performance?
* **Python‑level exception plumbing** – every call wraps everything in two nested `try/except` blocks.
* **Unnecessary data gymnastics** – converting between lists/dicts back‑and‑forth.
* **Repeated casting** – passing raw Python lists forces NumPy/Cython code inside SciPy to cast them again.
* **Logging** – every log message incurs a string format operation even if it never gets emitted.
  
If I strip those layers away I’m left with essentially one line of work per problem instance:

```python
transformed = scipy.ndimage.affine_transform(
    img_arr,
    mat_arr,
    order=3,
    mode="constant",
)
```

That line alone does all heavy lifting via BLAS‑backed routines inside SciPy’s compiled C code – there are no Python loops left at all after my tiny wrapper removes the outer exception handling and unnecessary conversions.
  
### Design decisions

1. **Input handling**  
   * Convert `problem["image"]` from a nested list into a NumPy array *once*, keeping it as a contiguous block (`np.asarray`).  
   * Do the same for `problem["matrix"]`.  
   * Use `dtype=np.float64` because SciPy’s internal kernels expect doubles – forcing an explicit cast avoids an implicit conversion during the actual transform.
   
   ```python
   img = np.asarray(problem["image"], dtype=np.float64)
   mat = np.asarray(problem["matrix"], dtype=np.float64)
   ```

2. **Affine call**  
   * Pass the full \(2\times3\) matrix straight through – this matches what the verifier does (`scipy.ndimage.affine_transform(image, matrix)`).
   * Keep default parameters (`order=3`, `mode="constant"`), matching reference behaviour.
   * Supply an explicit `cval=0` so constant padding is guaranteed even if defaults change.
   
   ```python
   out = scipy.ndimage.affine_transform(
       img,
       mat,
       order=3,
       mode="constant",
       cval=0,
   )
   ```

3. **Graceful failure**  
   * Wrap *only* the heavy transform in a narrow `try/except`.  
   * On any exception return an empty list under `"transformed_image"`.  
     This mirrors the original fallback path without paying the cost of catching every earlier step.
     
     ```python
     try:
         out = ...
     except Exception:
         return {"transformed_image": []}
     ```
