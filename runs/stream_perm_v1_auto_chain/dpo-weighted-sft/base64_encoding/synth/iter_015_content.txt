import base64; import numpy as np; from typing import Any, Dict

def solve(problem: Dict[str, Any]) -> str:
    data = problem.get('data')
    if data is None:
        return ''
    # Convert to numpy array
    if isinstance(data, (bytes, bytearray)):
        arr = np.frombuffer(data,dtype=np.uint8)
    else:
        arr = np.asarray(data,dtype=np.uint8)
    n = arr.size
    if n == 0:
        return ''
    # Pad
    pad_len = (-n) % 3
    if pad_len:
        arr_padded = np.concatenate([arr, np.zeros(pad_len,dtype=np.uint8)])
    else:
        arr_padded = arr
    # reshape
    groups = arr_padded.reshape(-1,3)
    # compute indices
    first = groups[:,0] >> 2
    second = ((groups[:,0] & 3) << 4) | (groups[:,1] >> 4)
    third = ((groups[:,1] & 15) << 2) | (groups[:,2] >> 6)
    fourth = groups[:,2] & 63
    indices = np.stack([first,second third,fourth],axis=1).astype(np.uint8)
    # lookup table
    alphabet_bytes = np.frombuffer(b'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/',dtype=np.uint8)
    encoded_bytes = alphabet_bytes[indices]
    encoded_flat = encoded_bytes.ravel()
    # add padding
    if pad_len ==1:
        encoded_flat[-2:] = ord('=')
    elif pad_len ==2:
        encoded_flat[-1:] = ord('=')
    return encoded_flat.tobytes().decode('ascii')

We need a fast Base64 encoder that stays pure‑Python‑free except for NumPy calls – the brief wants no Python‑level loops at all and full vectorisation.  
The classic Base64 algorithm turns every three input bytes into four ASCII characters by extracting six‑bit groups:

```
b00 b01 b02   →   c00 c01 c02 c03
```

where each `c` is an index into the standard alphabet  
`ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/`.  
The only tricky part is the padding when the input length isn’t a multiple of three – the output must end with `=` characters so that the total number of Base64 characters is a multiple of four.



### Data ingestion

The `problem` dict will contain a key `data` that holds either a `bytes`/`bytearray` object or an iterable of integers in `[0,255]`.

### Padding strategy
