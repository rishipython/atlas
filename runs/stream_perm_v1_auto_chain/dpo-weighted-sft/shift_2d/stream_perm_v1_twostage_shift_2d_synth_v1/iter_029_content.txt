import logging
import random
from typing import Any, Dict
import numpy as np
import scipy.ndimage

random.seed(42)
logging.basicConfig(level=logging.INFO)

class Shift2D:
    def solve(self, problem: Dict[str, Any]) -> Any:
        img = problem['image']
        shift = problem.get('shift', (0., 0.))
        mode = problem.get('mode', 'constant')
        cval = problem.get('cval', 0)
        order = problem.get('order', 3)

        # ensure numpy array
        arr = np.asarray(img)
        orig_dtype = arr.dtype

        # perform shift
        shifted = scipy.ndimage.shift(arr.astype(np.float64), shift=shift,
                                      order=order,
                                      mode=mode,
                                      cval=cval)

        # cast back preserving range
        if np.issubdtype(orig_dtype, np.integer):
            # clip before casting
            info = np.iinfo(orig_dtype)
            shifted_clipped = np.clip(np.round(shifted), info.min, info.max)
            return shifted_clipped.astype(orig_dtype)
        else:
            return shifted.astype(orig_dtype)

`ndimage.shift` returns a floating‑point array even when given an integer input because it needs room for interpolation coefficients that can temporarily exceed the integer bounds during computation.  
Therefore after shifting I’ll:
1. Round the result (`np.round`) because spline interpolation produces floats close to integers when shifting by an integer amount.
2. Clip it into the valid range of the original dtype (`np.clip(... , info.min , info.max)`).
3. Cast it back (`astype(orig_dtype)`).

For floating‑point inputs I’ll simply cast back without clipping – there’s no danger of overflow there anyway except perhaps tiny numerical noise which will be harmless when rounded back anyway.

I also have to keep in mind that SciPy’s `ndimage.shift` accepts either a scalar or a sequence for `shift`. In our case we always pass a two‑tuple `(dx, dy)`. The function handles negative values correctly so there’s nothing special needed there.
