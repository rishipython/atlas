We need a fast way to find the best 2‑D translation between two images without looping over every possible offset.
Let’s think about how the reference implementation probably works.
It probably slides one array over the other,
computes an error metric (sum of squared differences),
and picks the offset that minimises it.
Doing this naively would involve two nested Python loops – unacceptable.
Instead we can exploit the convolution theorem:
the cross‑correlation between two real images is just an inverse FFT of the product of one FFT with the complex conjugate of the other.
That gives us a full correlation surface in a single BLAS call.
From that surface we can locate the maximum – that’s our integer‑pixel best alignment.
If the task asks for sub‑pixel accuracy we can refine around the peak with a cheap centre‑of‑mass calculation on a tiny neighbourhood.
All this stays completely vectorised – no Python loops at all – and relies heavily on SciPy’s fast FFT routines.
The only place where SciPy really shows up is in `scipy.ndimage.shift` if we want to return an aligned copy of one image,
but even that call is just a thin wrapper over highly optimised C code.
We’ll also keep an eye on edge cases:
* Very small images – then there isn’t enough room for a 3×3 window;
* Zero‑sum windows – then centre‑of‑mass would blow up – just skip refinement;
* Wrap‑around behaviour – after finding the index of the maximum we must translate it into a signed displacement,
taking care that indices larger than half the dimension correspond to negative shifts because correlation is cyclic.
The algorithm below follows exactly this plan.
