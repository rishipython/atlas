The first thing I notice is that the baseline probably reads a dictionary that contains a key (the secret) and a collection of payloads that must be authenticated and then encoded in Base‑64. The simplest way to do this without writing any loops ourselves is to delegate the heavy lifting entirely to C‑level routines:

* `hmac.new` does the hashing in C.
* `base64.b64encode` does the encoding in C.
* `numpy.vectorize` lets us apply a Python callable over an array without writing an explicit loop – it still calls into C under the hood for each element but from our perspective there’s no Python‑level iteration.

With that in mind I’ll sketch out the structure:

```python
import base64
import hmac
import os
from typing import Any, Dict

# We'll bring NumPy up front so we don't have an import inside the method body.
import numpy as np

class Base64Encoding:
    def solve(self, problem: Dict[str, Any]) -> Dict[str, Any]:
        # Grab the secret key – fall back to 32‑byte random data if it isn’t supplied.
        raw_key = problem.get("key")
        if raw_key is None:
            key_bytes = os.urandom(32)
        else:
            # Assume callers pass a UTF‑8 string; encode it once.
            key_bytes = raw_key.encode()

        # Pull out whatever needs encoding – defaulting gracefully.
        payloads = problem.get("payloads", [])
        if not payloads:
            return {"results": []}

        # Vectorise a small lambda that does one round of HMAC + Base‑64.
        # Using np.vectorize keeps us out of Python loops while still being readable.
        encode_one = np.vectorize(
            lambda s: base64.b64encode(
                hmac.new(key_bytes, s.encode(), "sha256").digest()
            ).decode()
        )

        # Apply over the whole list and turn back into a plain Python list.
        results = encode_one(payloads).tolist()
        return {"results": results}
```

def run_solver(problem):
