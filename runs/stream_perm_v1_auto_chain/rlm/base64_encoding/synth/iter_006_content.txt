        data = problem.get('data')
        if isinstance(data,str):
            data = data.encode()
        elif isinstance(data,list):

        elif isinstance(data,np.ndarray):

        # flatten if multi-dimensional?

        n=data_flat.size
        if n==0:

        pad_len=(3-n%3)%3
        if pad_len:

We need a fast Base‑64 encoder that works on the dictionary format used by AlgoTune without any explicit Python loops inside the core logic.  
The simplest way is to treat the whole payload as a flat `uint8` array and perform the classic three‑byte‑to‑four‑character mapping in one go with NumPy’s vectorised arithmetic – all the heavy lifting happens in compiled C code under the hood and we never touch a `for` loop in Python itself.



**Key points**

* The input comes in `problem['data']`. It can be a string (`str`), a `bytes` object,
or an iterable/array of integers representing raw octets.
* Convert everything into a one‑dimensional `np.ndarray` of type `uint8`.  
  This guarantees that subsequent bit‑wise operations work on unsigned integers.
* Pad the stream so its length becomes a multiple of three – this is exactly what the standard algorithm does when it appends `=` characters at the end.
* Pack each consecutive triplet into a single 24‑bit integer (`uint32`) and split it back into four six‑bit indices (`i0 … i3`).  
  All shifts are performed after casting the triplets to `uint32` so that overflow does not occur.
* Use a pre‑computed lookup table containing the ASCII codes of the Base‑64 alphabet (`ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/`).  
  Indexing this table with the four index arrays yields the raw byte representation of all output characters at once.
* Finally overwrite the trailing characters corresponding to padding zeros with `'='`.  
  This step also runs in pure NumPy and touches only a handful of elements regardless of input size.
