Once we have the data in a `np.ndarray` of type `uint8`, the next step is the actual encoding logic:

* Pad the data so its length is a multiple of three – this guarantees that every group of three source bytes maps cleanly onto four destination characters.
* Split the padded array into three separate views (`r0`, `r1`, `r2`) using stride tricks (`b[::3]`, `b[1::3]`, `b[2::3]`).  
* Compute the four six‑bit indices for each group using bit‑shift and mask operations – all vectorised.
* Map those indices onto the standard Base‑64 alphabet (`A–Z`, `a–z`, `0–9`, `+`, `/`).  
* Finally apply any required “=`” padding back onto the result – this is done by overwriting the last one or two characters in place rather than creating new arrays again.

The alphabet itself can live as a global constant:
