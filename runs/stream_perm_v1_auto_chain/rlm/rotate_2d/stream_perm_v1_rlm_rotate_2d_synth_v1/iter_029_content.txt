We need to take the incoming `problem` dictionary and produce a rotated version of whatever 2‑D data it carries. The reference implementation is slow because it probably loops over every pixel or point and applies a hand‑rolled rotation formula. We can ditch that entirely by leveraging SciPy’s built‑in `ndimage.rotate`, which is written in C and already vectorised over the whole array. That satisfies the “no Python‑level loops” rule and gives us a huge speed boost.

First thought: just call `scipy.ndimage.rotate` on the input array with the supplied angle. That’s the simplest thing that works for images or any 2‑D array of floats or ints. But we should be careful about a few things:

- The input could be any NumPy‑compatible array, so we’ll cast it with `np.asarray` to be safe.
- The caller might expect the output shape to stay the same as the input (common for image processing pipelines). SciPy’s `rotate` has a `reshape` flag – if `False` it keeps the original shape and crops/pads as needed; if `True` it expands the output so nothing gets clipped.
- The rotation order (interpolation) matters for quality vs speed trade‑off. The baseline likely used nearest‑neighbour or bilinear interpolation; we’ll go with cubic (`order=3`) because it’s still fast enough and gives good visual results without being too heavy.
- Mode handling: default `'constant'` with `cval=0.0` matches many image rotation use‑cases where out‑of‑bounds pixels are set to zero.
- Angle can be negative or larger than 360 – SciPy handles that internally.
- We should expose an optional `reshape` key in the problem dict so callers can override the default behaviour if they want.
- No explicit loops anywhere – all operations are delegated to SciPy/C code.

With that in mind, the minimal skeleton looks like:

```python
from __future__ import annotations
from typing import Any, Dict
import numpy as np
import scipy.ndimage

def run_solver(problem: Dict[str, Any]) -> np.ndarray:
    img = np.asarray(problem["data"])
    angle = float(problem["angle"])
    reshape = bool(problem.get("reshape", True))
    return scipy.ndimage.rotate(
        img,
        angle,
        reshape=reshape,
        order=3,
        mode="constant",
        cval=0.0,
    )
```

That satisfies all constraints:

- Uses only NumPy and SciPy (specifically `scipy.ndimage`).
- No Python loops.
- Fully vectorised.
- Handles typical edge cases (empty arrays, zero angles).
- Keeps the same function name and signature expected by the test harness.
