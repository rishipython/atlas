### High‑level algorithm

1. **Early exit** – if `angle % 360 == 0` just copy the image.
2. **Build an affine transformation** – compute a 2×2 rotation matrix `R`.  
   The inverse transformation `R⁻¹` is simply its transpose because rotations are orthogonal.
3. **Generate an output coordinate grid** – create two 2‑D arrays containing all y‑coordinates (`Y`) and x‑coordinates (`X`) relative to the image centre.
4. **Map output coordinates back into input space** – apply `R⁻¹` via an explicit BLAS‐backed dot product (`np.dot`).  
   This satisfies the “matrix multiplication” requirement.
5. **Sample input image at mapped coordinates** – use `scipy.ndimage.map_coordinates`, which performs trilinear interpolation in C/C++ under the hood.
6. **Return the rotated array** – preserving dtype and shape.

All steps above are pure NumPy / SciPy calls – no Python loops except for trivial argument handling.

### Detailed design decisions & alternatives

#### Alternative A – `scipy.ndimage.rotate`
The simplest route would be:

Pros:

#### Alternative B – Manual affine transform + `map_coordinates`

### Edge cases & numerical stability

* **Zero / full‑turn angles** – handled by early exit (`angle % 360 == 0`). Avoids unnecessary computation.
* **Negative angles** – cosine stays positive/negative accordingly; sine flips sign automatically when computing `np.deg2rad(angle)`.
* **Non‑square images** – centre calculation works regardless of width vs height.
* **Large angles (>360°)** – modulo reduction isn’t strictly necessary because trig functions wrap naturally, but doing it keeps intermediate values small and avoids floating point drift when converting degrees → radians repeatedly.
* **Floating point precision** – using float64 everywhere ensures negligible rounding error compared with typical imaging tolerances.
* **Interpolation boundaries** – `mode='constant', cval=0` matches most baseline implementations that fill out‑of‑bounds pixels with zeros rather than wrapping or reflecting.
* **Memory consumption** – flattening grids into one dimension keeps memory contiguous and lets NumPy/BLAS operate efficiently on large contiguous blocks.
