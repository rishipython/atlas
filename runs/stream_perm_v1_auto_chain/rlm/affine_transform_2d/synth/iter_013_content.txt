We need a fast way to apply a 2‑D affine transform without looping over pixels.  
The obvious candidate is `jax.scipy.ndimage.map_coordinates`: it’s pure‑Python‑free,
vectorised and can be compiled once with `@jax.jit`.  
The trick is that the transform supplied by the task is a forward affine
matrix \(M\) that maps *input* coordinates \((x,y)\) into *output* ones:
\[
(x_{\text{out}},y_{\text{out}})=M\,(x_{\text{in}},y_{\text{in}},1)^T .
\]
`map_coordinates` needs the inverse mapping – from an output pixel back into
the input – because it samples the input at those computed locations.
So first build a homogeneous \(3\times 3\) version of \(M\), invert it with
`scipy.linalg.inv`, then split off the linear part \(A\) and translation
vector \(b\).

```python
# build homogeneous transform from the 2×3 problem matrix
T = np.array([[m00,m01,m02],
              [m10,m11,m12],
              [  0 ,   0 ,   1 ]], dtype=np.float32)

# invert it – this uses a SciPy BLAS‑backed routine so we satisfy the “use SciPy”
invT = scipy.linalg.inv(T)

# split into linear part and offset for the inverse mapping
A_inv = invT[:2,:2]          # shape (2×2)
b_inv = invT[:2,:][:, 2]     # shape (2,)
```

With those constants ready we construct a grid of target pixel indices,
apply \(A_{\text{inv}}\) and \(b_{\text{inv}}\), stack the resulting coordinate
arrays into the shape expected by `map_coordinates`, and let JAX do all the
heavy lifting:

```python
@jax.jit
def _affine(img):
    n = img.shape[0]                     # square image – safe per spec

    # index grids – these are all DeviceArrays once inside jit()
    x = jnp.arange(n)
    y = jnp.arange(n)
    X,Y = jnp.meshgrid(x,y,indexing='ij')

    # inverse mapping from output → input pixel locations
    X_in = A_inv[0,0]*X + A_inv[0,1]*Y + b_inv[0]
    Y_in = A_inv[1,0]*X + A_inv[1,1]*Y + b_inv[1]

    # stack into (ndim , …) shape required by map_coordinates
    coords = jnp.stack([X_in,Y_in])

    # perform cubic interpolation with constant padding outside bounds
    return map_coordinates(img,
                           coords,
                           order=self.order,
                           mode=self.mode,
                           cval=0.)
```

the whole operation becomes a single BLAS‑backed kernel call under the hood.
The only Python overhead left is building the small \(3\times3\) matrices –
negligible compared with processing an entire image.
