```
import logging
from typing import Dict
import numpy as np
import scipy.ndimage

class AffineTransform2D:
    def __init__(self):
        self.order=3
        self.mode='constant'

    def solve(self,p):
        try:
            img=np.asarray(p['image'],dtype=np.float64)
            mat=np.asarray(p['matrix'],dtype=np.float64)
            n=img.shape[0]
            # compute inverse
            M=mat[:,:2]
            t=mat[:,2]
            M_inv=np.linalg.inv(M)
            # create grid
            y,x=np.indices((n,n))
            # compute source coords
            src_y=M_inv[0][0]*(x - t[0]) + M_inv[0][1]*(y - t[1])
            src_x=M_inv[1][0]*(x - t[0]) + M_inv[1][1]*(y - t[1])
            # flatten
            src_coords=np.vstack((src_y.ravel(), src_x.ravel()))
            transformed_flat=scipy.ndimage.map_coordinates(img,src_coords,
                order=self.order,
                mode=self.mode,
                cval=0.0,
                prefilter=True)
            transformed_img=transformed_flat.reshape((n,n))
            return {"transformed_image": transformed_img}
        except Exception as e:
            logging.error(f"Error in solve: {e}")
            return {"transformed_image": []}
```

The simplest way is still to do an inverse‑map based sampling like `affine_transform`, but instead of letting `affine_transform` build all the coordinate grids itself we build them once in NumPy and hand them over to `scipy.ndimage.map_coordinates`.  
`map_coordinates` does the heavy lifting in C – it takes a set of source coordinates and interpolates the image at those points – so there’s no Python‑level loop over pixels any more.  
The key steps are:

* Parse the input dictionary – pull out the image (`float64` for safety) and the 2×3 affine matrix.
* Compute the inverse linear part (`M⁻¹`) once (`M` is the first two columns).
* Build a full grid of destination indices (`np.indices`) – this gives us two integer arrays `y,x` shaped `(n,n)` where `n` is the side length.
* Transform these indices back into source coordinates:
  ```
  src_y = M_inv[0][0]*(x-t[0]) + M_inv[0][1]*(y-t[1])
  src_x = M_inv[1][0]*(x-t[0]) + M_inv[1][1]*(y-t[1])
  ```
  Here `t` are the translation components from the original matrix.
* Flatten those two coordinate arrays into a single `(2,n*n)` array (`np.vstack([src_y.ravel(), src_x.ravel()])`) – this matches what `map_coordinates` expects.
* Call `scipy.ndimage.map_coordinates` once:
