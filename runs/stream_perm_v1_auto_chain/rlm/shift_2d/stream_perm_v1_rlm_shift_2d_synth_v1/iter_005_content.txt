We need to write a fast `run_solver` that does the same thing as the slow `Shift2D().solve(problem)` but without any Python loops and making heavy use of SciPy’s fast routines.  
The problem is almost certainly a “find the best 2‑D integer shift that aligns two images” task – that’s what the name *shift_2d* hints at and what the reference implementation does internally (it probably builds a correlation surface and picks the peak).

The simplest way to get a correlation surface in full vectorised form is an FFT‑based cross‑correlation:
```
corr = ifft2( fft(x) * conj(fft(y)) )
```
That gives us an array of the same shape as `x` (and `y`) where each element tells us how well `y` matches `x` when shifted by that offset (modulo wrap‑around).  
The peak of this surface is exactly the best integer alignment – we just have to convert the index of that peak into a signed shift amount (negative indices mean “shift left/up”).

Once we have the optimal `(dy, dx)` pair we can apply it to `y` using SciPy’s `ndimage.shift`.  That routine handles sub‑pixel interpolation if we want it – but for an integer alignment we could just roll the array; using `ndimage.shift` keeps the interface simple and still stays vectorised because it works on whole arrays at once.

- **Input format** – The test harness will give us a dictionary with at least two keys (`'x'` and `'y'`) containing NumPy arrays of identical shape (most likely float32/float64).  
- **Output format** – The baseline returns a dictionary containing at least `'shift'` (a list `[dy, dx]`) and `'aligned'` (the shifted version of `y`).  Mimicking that keeps compatibility.
- **Precision** – Work in float64 internally so rounding errors are minimal; cast back to float32 only if necessary.
- **Edge cases** – If either array is empty or has zero size we just return zero shift and an empty array.

Putting it together:

```python
import logging
from typing import Any, Dict
import numpy as np
import scipy.ndimage

def run_solver(problem: Dict[str, Any]) -> Dict[str, Any]:
    """
    Solve the 2‑D shift alignment problem.
    """
    # Grab inputs
    x = np.asarray(problem['x'])
    y = np.asarray(problem['y'])
    
    # Quick sanity checks
    if x.size == 0 or y.size == 0:
        return {'shift': [0, 0], 'aligned': y}
    
    # Ensure same shape
    assert x.shape == y.shape
    
    # Compute cross‑correlation via FFT
    Xf = np.fft.rfftn(x)
    Yf = np.fft.rfftn(y)
    corr = np.fft.irfftn(Xf * np.conj(Yf), s=x.shape)
    
    # Find peak index
    peak_idx = np.unravel_index(np.argmax(corr), corr.shape)
    
    # Convert peak index into signed shifts
    mid_y, mid_x = (np.array(x.shape) // 2)
    dy = peak_idx[0] - mid_y
    dx = peak_idx[1] - mid_x
    
    # Apply the found shift to y
    aligned_y = scipy.ndimage.shift(
        y,
        shift=(-dy, -dx),
        order=1,
        mode='constant',
        cval=0,
        prefilter=False,
        output=None,
        axes=None,
        clip=False,
        extra_args=(),
        extra_keywords={}
    )
    
    return {
        'shift': [int(dy), int(dx)],
        'aligned': aligned_y.astype(y.dtype)
    }
```

This implementation:

* Uses only NumPy and SciPy – no explicit Python loops.
* Relies on highly optimised FFTs for correlation.
* Uses SciPy’s efficient interpolation routine for applying the final shift.
* Keeps everything vectorised and should run in milliseconds on typical image sizes while matching the expected output format of the reference solver.
