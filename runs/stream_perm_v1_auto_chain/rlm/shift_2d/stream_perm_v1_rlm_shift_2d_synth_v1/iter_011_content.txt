We need to take the incoming `problem` dictionary and produce a shifted 2‑D array as fast as possible.  
The reference `Shift2D().solve(problem)` is slow because it probably loops over every pixel or uses a pure‑Python implementation of the FFT‑based cross‑correlation routine.  The quickest path is to hand off the heavy lifting to SciPy’s highly optimised C/Fortran kernels.

First thought: just call `scipy.ndimage.shift`.  That would satisfy the “no Python loops” rule and use BLAS/LAPACK under the hood.  But we have to be careful about the data layout: SciPy expects a tuple `(shift_y, shift_x)` where positive values move the array towards higher indices (i.e., down/right).  The problem specification might give shifts in the opposite direction or as pixel offsets that include sub‑pixel components.  We’ll normalise that by flipping signs if needed and by converting everything to float64 before handing it off.

Second thought: what if the problem wants us to find the optimal integer shift that maximises correlation between two images?  In that case a brute‑force loop over all candidate shifts would be too slow.  Instead we can compute the cross‑correlation via FFT once and then pick the argmax of the resulting matrix – all vectorised in NumPy.  This still uses only SciPy/NumPy calls and no explicit Python loops.

Let’s sketch both possibilities:

1. **Direct shift** – `problem` contains keys:
   - `'array'`: a 2‑D NumPy array
   - `'shift'`: tuple/list of floats `(dx, dy)`
   The output is simply the shifted array.

2. **Optimal integer shift** – `problem` contains keys:
   - `'source'`: first image
   - `'target'`: second image
   The solver should return an integer `(dx, dy)` that maximises cross‑correlation and also provide the shifted source image for verification.

We’ll detect which mode we’re in by checking for presence of `'source'` vs `'array'`.  That keeps the interface flexible while still meeting the “single function name” requirement.

Now about numerical precision:  
- Use `np.float64` everywhere – SciPy’s FFT functions are most stable in double precision.
- When computing correlation via FFT we should subtract mean from each image first so that zero‑mean signals give better correlation peaks.
- After finding the peak location we must correct for wrap‑around because FFT cross‑correlation is circular.  The true linear correlation peak is at `(peak_y - H + 1 if peak_y >= H//2 else peak_y,
    peak_x - W + 1 if peak_x >= W//2 else peak_x)` where `H,W` are image heights/widths.  
- Clip any tiny negative values from numerical noise when computing power spectra before taking square roots (though this isn’t strictly needed for correlation).

- Use `scipy.signal.ifftn` / `fftn` instead of higher level wrappers for speed.

```python
def _normalize_shift(shift):
    s = np.asarray(shift, dtype=np.float64)
    if s.shape != (2,) and s.shape != (1,):
        raise ValueError("Shift must be a scalar or pair.")
    if s.shape == (1,):
        s = np.array([s[0], s[0]], dtype=np.float64)
    # SciPy expects (dy, dx)
    return (float(s[0]), float(s[1]))
```

Direct shift:

```python
def _direct_shift(arr: np.ndarray, shift):
    logger.debug(f"Shifting array shape {arr.shape} by {shift}")
    return scipy.ndimage.shift(
        arr,
        shift=shift,
        order=3,
        mode='constant',
        cval=0.0,
        prefilter=True,
        output=None,
        axes=(0,1)
    )
```

Wait — that seems wrong; we want convolution not elementwise multiplication. Actually cross-correlation via FFT is computed as inverse FFT of product of one FFT times conjugate of other:

```python
src_fft = scipy.signal.fftn(src)
tgt_fft = scipy.signal.fftn(tgt)
corr_fft = src_fft * np.conj(tgt_fft)
corr = scipy.signal.ifftn(corr_fft).real
```

But due to circularity we must pad appropriately or use `scipy.signal.correlate2d` which handles linear correlation internally but slower. We can pad both arrays with zeros up to size H+H-1 x W+W-1 before FFT:

```python
pad_H = H + H - 1
pad_W = W + W - 1
src_pad = np.zeros((pad_H,pad_W), dtype=np.float64)
tgt_pad = np.zeros((pad_H,pad_W), dtype=np.float64)
src_pad[:H,:W] = src
tgt_pad[:H,:W] = tgt

src_fft = scipy.signal.fftn(src_pad)
tgt_fft = scipy.signal.fftn(tgt_pad)
corr_fft = src_fft * np.conj(tgt_fft)
corr = scipy.signal.ifftn(corr_fft).real

# Find peak index
iy,iX = np.unravel_index(np.argmax(corr), corr.shape)

# Convert linear offset back to integer shifts relative to original size:
dy = iy - (H - 1)
dx = iX - (W - 1)

return (dx, dy), _direct_shift(source,(dy,dx))
```
